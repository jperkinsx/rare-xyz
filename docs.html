Rare Protocol

The open marketplace protocol for NFTs. Onchain auctions, agent-native tooling, $RARE burns.

Rare Protocol is a permissionless NFT marketplace protocol deployed on Ethereum. It provides onchain timed auctions, fixed-price sales, offers, and a fee architecture powered by the $RARE token. Anyone can interact with the contracts directly — no frontend required.

Sepolia Testnet — The protocol is currently deployed on Sepolia for testing. All contracts are verified and open source.

Architecture

The protocol consists of five core contracts:

  • SuperRareBazaar — The marketplace hub. Handles sales, offers, auctions, and settlement. All trade flows route through here.
  • RareMinter — Managed minting with configurable direct sale parameters, allowlists, and split payments.
  • SovereignNFTContractFactory — Deploys new ERC-721 collections with ERC-2981 royalties. Each collection is its own contract, not a shared one.
  • RareAppRegistry — Apps register here with a fee in basis points. The protocol takes a revenue share.
  • $RARE Token — ERC-20 governance token. Protocol fee revenue is used to buy and burn $RARE.

Timed Auctions

Rare Protocol is the only NFT protocol with fully onchain timed auctions. Three auction types are supported:

Reserve Auction (Coldie)

The auction clock starts when the first bid meets the minimum. If no one bids, the auction sits dormant. Once triggered, it runs for the configured duration with anti-snipe extensions on late bids.

Scheduled Auction

Set a future start time. The auction begins automatically at that timestamp and runs for the configured duration. Useful for coordinated drops.

Anti-Snipe Extension

Bids placed in the final minutes of an auction extend the clock. This prevents last-second sniping and ensures fair price discovery. The extension length is configurable by the protocol admin.

Settlement

After the auction ends, anyone can call settleAuction() to transfer the NFT to the winner and pay the seller. Settlement is permissionless — no relayer needed.

Fee Structure

The fee architecture is designed around zero friction for casual users and revenue sharing for apps:

  • No app (_app = address(0)) — Zero protocol fees. Just gas and creator royalties (ERC-2981).
  • Registered app — The app's fee (set in basis points) is deducted from the sale. The protocol takes a configurable share (default 20%) of the app fee.

Fee calculation: On a 1 ETH sale through an app with 250 bps (2.5%) fee and 20% protocol share:

Fee breakdown
Sale amount:        1.0 ETH
Creator royalty:    0.1 ETH  (10% via ERC-2981)
App fee:            0.025 ETH (2.5% of sale)
  App keeps:        0.020 ETH (80% of app fee)
  Protocol share:   0.005 ETH (20% of app fee → burned as $RARE)
Seller receives:    0.875 ETH

$RARE Token Burns

Protocol revenue (the protocol's share of app fees) accumulates in the network beneficiary address. This revenue is periodically used to:

  1. Buy $RARE on the open market
  2. Burn the purchased $RARE, permanently removing it from supply

Burn rate scales directly with marketplace volume. More sales through registered apps means more $RARE burned. Direct/casual usage (with no app) generates zero protocol revenue and zero burns — this is by design to keep the casual path frictionless.

App Registry

Any address can register as a marketplace app:

Register an app
// Register with a 2.5% fee (250 basis points)
appRegistry.registerApp(250);

// Update fee later
appRegistry.setAppFee(300); // now 3%

// Check an app's fee
uint16 fee = appRegistry.getAppFee(appAddress);

When users trade through your app, pass your address as the _app parameter in Bazaar functions (setSalePrice, offer, configureAuction). Your fee is automatically deducted and split with the protocol.

Agent Skill Installation

Add the Rare Protocol skill to your AI agent to enable NFT minting and auction management. The skill provides a complete set of marketplace commands that agents can execute directly against the Sepolia contracts.

Installation

skill-config.yaml
name: rare-protocol
version: 2.4.1
description: Mint, list, auction, and trade NFTs on Rare Protocol
network: sepolia
contracts:
  bazaar: 0xCE999aeF38E1316A510615b80713b494658807c6
  minter: 0xf689b3344090DdE8b7a2C6b942730fF05De29451
  factory: 0xeF4aac03af2684bE0BE2B07c451949CC4246085F
  appRegistry: 0x3ad7694F899804206076F9A8E6F06719E360A90b

Capabilities

Once installed, your agent can:

  • Deploy collections — Create new ERC-721 contracts with custom name and symbol
  • Mint NFTs — Mint tokens with IPFS metadata and ERC-2981 royalties
  • Fixed-price sales — List tokens, buy listed tokens, delist
  • Timed auctions — Create reserve/scheduled auctions, place bids, settle
  • Offers — Place offers, cancel, accept (with optional convertible-to-auction flag)
  • Query state — Read token ownership, sale prices, auction details, app fees
  • App management — Register as an app, set fees, query fee info

Example: Agent-driven minting flow

Natural language prompt
"Deploy a new collection called 'Dreamscapes' with symbol DRMS.
 Then mint 3 tokens using these IPFS URIs:
   - ipfs://QmHash1
   - ipfs://QmHash2
   - ipfs://QmHash3
 List #1 for 0.5 ETH and create a reserve auction
 for #2 with a 0.1 ETH minimum bid."

The agent will execute the full sequence: factory deploy, 3x mintTo, setSalePrice, configureAuction — each as a separate transaction with wallet confirmation.

CLI Reference

The web CLI provides a terminal interface for all protocol operations. Connect your wallet and type commands directly.

Wallet

connect              Connect MetaMask, switch to Sepolia
status               Show address, balance, network
contracts            List all deployed contract addresses

Minting

mint deploy --name "Name" --symbol SYM       Deploy collection
mint token  --contract 0x... --uri ipfs://...  Mint NFT
mint info   --contract 0x...                   Collection info

Marketplace

list   --contract 0x... --id 1 --price 0.5    List for sale
buy    --contract 0x... --id 1 --price 0.5    Buy listed token
delist --contract 0x... --id 1              Remove listing

Auctions

auction create  --contract 0x... --id 1 --min 0.1 --duration 3600
auction bid     --contract 0x... --id 1 --amount 0.2
auction settle  --contract 0x... --id 1
auction cancel  --contract 0x... --id 1
auction info    --contract 0x... --id 1

Offers

offer place  --contract 0x... --id 1 --amount 0.3
offer cancel --contract 0x... --id 1

Apps

apps register --fee 250           Register (250 bps = 2.5%)
apps set-fee  --fee 300           Update fee
apps info     --app 0x...         Look up app fee

Indexer API

The protocol is indexed by an Envio indexer that tracks all marketplace events in real time.

GraphQL Endpoint
https://indexer.dev.hyperindex.xyz/d59a89b/v1/graphql

The indexer tracks: sales, listings, offers, auctions, bids, settlements, mints, collection deployments, and app registrations. It also fetches IPFS metadata and resolves auction state onchain via Effects.
